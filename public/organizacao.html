Sistema de Organização do Squad

Arquitetura de ferramentas otimizada para integração máxima com Claude

Veredicto: Linear + Notion + GitHub + n8n

Depois de analisar as ferramentas que vocês já têm conectadas, a resposta é clara. Vocês não precisam de mais nenhuma ferramenta — precisam usar o que já têm da forma certa.

A combinação Linear + Notion + GitHub é a mais poderosa que vocês poderiam ter porque as três estão diretamente integradas ao Claude via MCP. Isso significa que o Claude pode criar issues, buscar documentação e interagir com os repos sem vocês saírem da conversa.

Por que não ClickUp/Obsidian/Trello? Nenhum deles tem integração MCP com o Claude. Cada minuto que vocês gastam fora do ecossistema integrado é produtividade perdida. O poder do setup de vocês é que o Claude pode "ver" tudo — issues, docs, workflows, deploys, banco — e agir sobre tudo de um lugar só.

Por que Linear + Notion (e não as alternativas)

Critério Linear + Notion ClickUp Obsidian Jira + Confluence
Integração MCP com Claude Ambos conectados Nenhuma Nenhuma Parcial (Jira só)
Claude cria/edita tasks Direto via MCP Não Não Limitado

Velocidade de uso Ultra rápido Lento/pesado Rápido (local) Muito lento

Arquitetura Completa do Ecossistema

Camada 1 — Comando Central (Claude)
Claude Max (2-3 planos)
Cérebro do squad. Acessa tudo abaixo via MCP. Coda, planeja, documenta, revisa.
↕ MCP (acesso direto bidirecional) ↕
Camada 2 — Gestão e Documentação
Linear
Issues, sprints, bugs, features. Cada projeto = 1 projeto Linear. Claude cria e atualiza issues.
Notion
Wiki do squad, SOPs, meeting notes, knowledge base, docs de produto, financeiro.
GitHub
Repos de código. 1 org, 1 repo por projeto. PRs, branches, CI/CD.
↕ Integrado ↕
Camada 3 — Infraestrutura e Produto
Supabase
Banco, auth, storage, edge functions. Claude executa SQL direto.
Vercel
Deploy dos frontends. Claude vê logs, builds e faz deploy.
Stripe
Pagamentos. Claude lista clientes, invoices, cria produtos e preços.
n8n
Automações: notificações, reports, webhooks. Claude busca e executa workflows.
↕
Camada 4 — Storage e Comunicação
Google Drive
Propostas comerciais, contratos, assets visuais. Claude busca e lê arquivos.
WhatsApp (Grupo)
Comunicação rápida do squad. Quick syncs, alertas. NÃO para decisões importantes.
O diferencial: Todas as ferramentas das camadas 1-3 estão conectadas ao Claude via MCP. Isso significa que vocês podem pedir "Claude, cria uma issue no Linear para o bug X, verifica o log do Vercel, e consulta a tabela Y no Supabase" — tudo numa conversa só. Nenhum squad no Brasil opera nesse nível de integração.

Estrutura no Linear — Gestão de Tarefas

Organização: 1 Team por Área de Negócio

O time "Ruralfy" que já existe vira só mais um. Criem teams separados para cada frente:

Labels padronizados para todo o workspace

  • revenue — Gera dinheiro diretamente
  • product — Constrói produto/ativo
  • infra — Setup, devops, tooling
  • blocked — Esperando algo externo
  • quick-win — Menos de 2h para concluir

Ciclos semanais (Sprints)

  • Sprint de segunda a sexta (5 dias)
  • Cada membro puxa no máximo 5-7 issues por sprint
  • Issues com label revenue sempre têm prioridade "Urgent"
  • Daily: cada um atualiza o status das suas issues no Linear (In Progress / Done)

Como o Claude interage

  • "Claude, cria uma issue no time Agência IA: implementar bot para Cliente X"
  • "Claude, quais issues estão In Progress no WhatsApp SaaS?"
  • "Claude, move a issue RUR-42 para Done e adiciona comentário de entrega"
  • "Claude, lista tudo que está bloqueado no squad essa semana"

Estrutura no Notion — Knowledge Base

Notion = Wiki + Docs + Decisões (NÃO tasks)

Regra fundamental: tasks ficam no Linear, documentação fica no Notion. Nunca misturem. O Notion serve para tudo que não é "o que fazer agora" — é o cérebro de longo prazo do squad.

Git Workflow: Trunk-Based (simplificado)

  • main = produção, sempre deployável
  • feature/xxx = branches curtas (máx 1-2 dias)
  • PR obrigatório para main, mas review rápido (Claude pode ajudar)
  • Sem develop/staging — vocês são 4, não precisam de cerimônia
  • Commits com prefixo: feat:, fix:, docs:, chore:

Integração Linear ↔ GitHub

  • Ativem a integração nativa Linear-GitHub
  • Branch name = prefixo da issue Linear (ex: feat/AGN-42-whatsapp-bot)
  • PR fecha a issue automaticamente quando mergeado

Checklist de Setup — Dia 1

O que configurar ANTES de começar a codar

Linear (30min)

  • Criar teams: Squad Ops, Agência IA, WhatsApp SaaS, CyberShield, Produtos Digitais
  • Criar labels globais: revenue, product, infra, blocked, quick-win
  • Configurar ciclos semanais (seg-sex)
  • Convidar os 4 membros
  • Integrar com GitHub (Settings → Integrations → GitHub)

Notion (1h)

  • Criar a estrutura de páginas conforme o mapa acima
  • Criar os databases: Financeiro, Pipeline Clientes, Decisões, Dailys
  • Copiar o plano estratégico para a página "Estratégia 30 Dias"
  • Criar template de daily (propriedades: data, participantes, progresso, blockers)
  • Convidar os 4 membros

GitHub (30min)

  • Criar org "squad-fiap" (ou nome que preferirem)
  • Criar repos iniciais: agency-templates, whatsapp-saas
  • Configurar branch protection em main (require PR)
  • Conectar Vercel aos repos

n8n (1h)

  • Criar workspace organizado por pastas
  • Montar a automação de Daily Report (prioridade 1)
  • Montar a automação de Revenue Tracker
  • Testar webhook de lead capture

Claude Max — Skills customizadas

  • Criar 1 skill por projeto (contexto persistente para cada produto)
  • Documentar stack técnica, decisões e padrões em cada skill
  • Compartilhar as skills entre as contas para consistência

Superpoderes — O que esse setup permite

"Claude, prepara meu dia"

Claude consulta suas issues no Linear, verifica deploys no Vercel, checa métricas no Supabase, e te dá um briefing completo do que fazer. Cada membro começa o dia com contexto total em 30 segundos.

"Claude, entrega esse projeto"

Claude lê a proposta no Google Drive, puxa o escopo das issues do Linear, monta o código baseado nos templates do GitHub, faz deploy no Vercel, configura o Stripe, e marca as issues como Done. Um projeto de agência entregue em horas.

"Claude, como estamos financeiramente?"

Claude consulta o Stripe (receita), Notion (custos registrados), e Linear (projetos em andamento) para te dar um snapshot financeiro completo. MRR, pipeline, burn rate — tudo em uma resposta.

"Claude, audita o site do cliente"

Henrique pede ao Claude para gerar os comandos de scan, processar os resultados, e gerar o relatório PDF de auditoria — tudo via uma conversa. De scan a relatório entregue em 1 hora em vez de 1 dia.
